## สรุปเนื้อหา

คลิปนี้เป็นการบรรยายเกี่ยวกับการพัฒนาซอฟต์แวร์ในศตวรรษที่ 21 โดยเน้นประเด็นสำคัญเรื่องโครงสร้างข้อมูลและ schema ในระบบซอฟต์แวร์ วิทยากรอธิบายว่าซอฟต์แวร์ในปัจจุบันมีความซับซ้อนมากขึ้น และต้องรองรับการเปลี่ยนแปลงอยู่ตลอดเวลา ดังนั้นการออกแบบ schema จึงมีผลอย่างมากต่อคุณภาพและความยั่งยืนของระบบ

โดยMartin Fowler อธิบายความแตกต่างระหว่าง explicit schema และ implicit schema ซึ่งพบได้บ่อยทั้งในฐานข้อมูลและการเขียนโปรแกรมในหน่วยความจำ (in-memory programming) โดย explicit schema คือการกำหนดโครงสร้างข้อมูลไว้อย่างชัดเจน เช่น ฐานข้อมูลเชิงสัมพันธ์ที่มีการกำหนดชนิดข้อมูล ฟิลด์ และความสัมพันธ์ระหว่างข้อมูลอย่างแน่นอน วิธีนี้ช่วยให้ระบบเข้าใจง่าย สามารถตรวจสอบความถูกต้องของข้อมูลได้ตั้งแต่ต้น และช่วยลดความผิดพลาดที่อาจเกิดขึ้นในระหว่างการพัฒนาและการใช้งาน

ในทางตรงกันข้าม implicit schema เป็นโครงสร้างข้อมูลที่ไม่ได้กำหนดรูปแบบไว้อย่างชัดเจน เช่น การใช้ dictionary หรือ JSON ซึ่งแม้จะมีความยืดหยุ่นสูงและเหมาะกับการพัฒนาที่ต้องการความรวดเร็ว แต่ก็อาจนำไปสู่ปัญหา เช่น ข้อมูลไม่สม่ำเสมอ การตรวจสอบข้อมูลตามบริบทที่ทำได้ยาก และความซับซ้อนในการดูแลและปรับปรุงระบบในระยะยาว

สรุปคลิปนี้แสดงให้เห็นว่าการออกแบบ schema เป็นส่วนสำคัญของการออกแบบระบบซอฟต์แวร์หากพึ่งพา implicit schema มากเกินไปโดยไม่มีการควบคุมหรือการตรวจสอบที่เหมาะสม อาจทำให้ระบบแก้ไขยาก เกิดข้อผิดพลาดได้ง่าย และส่งผลต่อคุณภาพของซอฟต์แวร์ในอนาคต
